# CONTEXT BUDGET — 예산 선언과 소진 장부

컨텍스트는 예산이다 (원칙 2).

## 정책: 보통 (80% 경고 / 100% 체크포인트+교체)

| 항목 | 값 |
|---|---|
| 예산 단위 | 작업 단위 수 (토큰 측정 불가 환경) |
| 작업 단위당 예산 | 세션 컨텍스트의 1/4 (개정 안건 1건 기준 — 초과 예상 시 안건 분할) |
| 80% 도달 시 | 체크포인트 → 현 단위만 마무리, 신규 착수 금지 |
| 100% 초과 시 | 즉시 체크포인트 → 잔여 작업 state.json 기록 → 세션 종료/교체 |
| 세션 시작 읽기 상한 | HARNESS.md + state.json + current.refs 파일만 |

## 하네스 평가 예산 예시

실제 값의 정본은 이 예제에서 생략한 `harness-spec.json.self_evaluation`이다.

| 항목 | 예시 값 | 규칙 |
|---|---:|---|
| evaluation budget ratio | 0.10 | 비필수 평가 비용이 운영 비용의 10% 이상이면 유예 |
| cooldown | 작업 단위 3개 | 완료된 평가 뒤 비필수 재평가 억제 |
| mandatory full | 예산·cooldown 무관 | 계약 변경·콜드스타트·반복 실패를 묵살하지 않음 |

평가 비용은 운영 비용과 별도 합산한다. checker와 recorder는 결정적 로컬 작업이며 LLM 평가 비용으로 부풀리지 않는다.

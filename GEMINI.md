# Harness Factory

Use the on-demand skills in `skills/` to build and evolve project-owned runtime-neutral harnesses.

- Start with `build-harness` for a full harness.
- Use `build-agent`, `build-skill`, or `build-evaluator` for one component.
- Use `verify-harness` for structure and adapter parity.
- Use `evaluate-harness` only after an explicit request or non-`none` deterministic trigger decision.
- Use `improve-harness` only from verified evaluation evidence.

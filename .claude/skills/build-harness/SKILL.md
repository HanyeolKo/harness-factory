# build-harness

harness-factory의 구성자 수행 프로토콜(README.md §3)을 실행한다.

## 절차

1. **Phase 0 — 원칙 로드**: 이 레포의 `README.md`, `principles/` 전체(01~06), `interview/QUESTION-BANK.md`를 읽는다. 대상 프로젝트가 지정됐으면 구조를 훑어 결정적 evaluator 후보(테스트/빌드/린트 수단)를 찾는다.
2. **Phase 1 — 인터뷰**: `interview/QUESTION-BANK.md`의 1차 배치 4문항을 `AskUserQuestion`으로 질의한다 (코드베이스에서 이미 확인한 답은 소거). 답변 분기에 따라 2차 배치를 최대 4문항 추가. 사용자가 "알아서"라고 하면 기본값 일괄표를 적용한다.
3. **Phase 2 — 결정 확정**: 답변을 템플릿 치환 필드에 매핑하고, 인터뷰 요약과 적용 기본값을 D-001 초안으로 정리한다.
4. **Phase 3 — 인스턴스화**: `templates/` 이하를 대상 위치(기본 `<대상>/harness/`)로 복사하며 `{{...}}`를 전부 치환한다. README §4 산출물 계약의 구조·파일명을 유지한다. state.json에 초기 큐와 `next_action`을 채운다. 미사용 축은 파일 내 "미사용 — 사유" 표기.
5. **Phase 4 — 검증·인도**: `CHECKLIST.md` 전 항목 수행. 특히 `grep -rn '{{' <대상>` 0건 확인과 콜드스타트 테스트(HARNESS.md만 읽고 3문항에 답하기)를 실제로 수행한다. 인도 보고(생성 트리 / 적용 결정 / 첫 실행 방법 / 첫 백로그)로 마친다.

## 인자 해석

- 경로가 주어지면: 해당 경로를 대상 프로젝트로 삼는다.
- 작업 설명이 주어지면: Q1(대상·산출물)의 답으로 선반영하고 인터뷰에서 재확인만 한다.
- 인자가 없으면: 현재 작업 디렉토리를 대상으로 Q1부터 질의한다.
